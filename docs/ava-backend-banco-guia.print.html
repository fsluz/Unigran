UNIGRAN CONFIGURACAO PRATICA SEM EXPERIENCIA TECNICA

Manual para configurar
dados e arquivos do AVA

TypeDB + Supabase Storage + Cloudinary

Passo a passo para colocar documentos, imagens, videos e portfolio funcionando sem criar banco desnecessario ou misturar responsabilidades.

Para quem: pessoa que recebeu as contas dos servicos e precisa configurar o projeto.

Escopo: AVA professor/aluno e portfolio academico.

Versao: 2.0 | Data: 25/05/2026

1. Antes de comecar

Regra simples: ha tres servicos e cada um faz uma coisa. Nao crie tabelas novas no Supabase para portfolio ou AVA nesta configuracao.
ServicoO que guardaExemplo visivel
TypeDBTextos, usuarios, posts, metadados e vinculos do portfolio.Titulo do projeto, descricao formatada, URL do case.
Supabase StorageSomente arquivos/documentos.PDF da entrega, DOCX, curriculo anexado.
CloudinarySomente imagens e videos.Capa do post, imagem do projeto, video Zuni.

Voce precisa receber antes

  • Acesso ao projeto Supabase correto, com permissao para Storage e SQL Editor.
  • Acesso ao painel Cloudinary ou as tres credenciais da conta.
  • Credenciais do cluster TypeDB ja usado pelo sistema.
  • Acesso ao projeto backend na Vercel para adicionar variaveis de ambiente.
  • Node.js instalado caso queira validar localmente.
Nunca compartilhe: SUPABASE_SERVICE_ROLE_KEY, CLOUDINARY_API_SECRET, TYPEDB_PASSWORD ou JWT_SECRET. Elas entram apenas no backend/Vercel, nunca em tela publica nem no frontend.

2. Supabase: configurar documentos

Use o Supabase apenas para guardar arquivos. O backend envia o arquivo e recebe uma URL publica para gravar junto aos dados do portfolio.

1 Abrir o projeto certo
  1. Entre em supabase.com e abra o projeto usado pelo UNIGRAN.
  2. No menu esquerdo, abra SQL Editor.
  3. Clique em New query.
2 Criar somente o bucket

No computador do projeto, abra o arquivo backend/supabase/002_ava_storage_only.sql. Copie todo o conteudo, cole no SQL Editor e clique em Run.

Nao execute para esta arquitetura: backend/supabase/001_portfolio_academico.sql. Esse arquivo antigo tambem cria tabelas no Supabase. Aqui a escrita estruturada pertence ao TypeDB.
3 Conferir resultado
  1. No menu esquerdo, abra Storage.
  2. Confirme que existe o bucket ava-entregas.
  3. Confirme que ele esta como Public. Isso permite abrir PDF pelo link do portfolio.
4 Copiar as credenciais
  1. Abra Project Settings > API.
  2. Copie o Project URL: vira SUPABASE_URL.
  3. Copie a chave service_role: vira SUPABASE_SERVICE_ROLE_KEY.
SUPABASE_URL=https://SEU_PROJECT_REF.supabase.co
SUPABASE_SERVICE_ROLE_KEY=COLE_A_CHAVE_SERVICE_ROLE
SUPABASE_DOCUMENTS_BUCKET=ava-entregas
Atencao: a chave service_role ignora politicas de seguranca. Coloque-a somente no ambiente do backend, nunca no arquivo frontend/.env.

3. Cloudinary: configurar imagens e videos

Cloudinary recebe as midias visuais. O sistema ja envia imagem/video para ele e grava a URL retornada no post.

1 Obter chaves
  1. Entre em cloudinary.com e abra o console da conta do projeto.
  2. No Dashboard ou em API Keys, localize Cloud name, API Key e API Secret.
  3. Copie cada valor para as variaveis do backend.
CLOUDINARY_CLOUD_NAME=seu_cloud_name
CLOUDINARY_API_KEY=sua_api_key
CLOUDINARY_API_SECRET=seu_api_secret
2 O que o sistema cria
Acao no portalPasta CloudinaryResultado
Post com imagem/videounigran/postsURL usada no feed e portfolio.
Video do Zuniunigran/postsVideo validado com limite configurado.
Comentario com midiaunigran/commentsMidia exibida na discussao.
3 Testar pela aplicacao
  1. Inicie frontend e backend conforme a secao 7.
  2. Entre com um usuario.
  3. Crie um post com uma imagem pequena.
  4. Se a imagem aparecer no card, Cloudinary esta configurado.
Erro comum: mensagem Cloudinary nao configurado significa que uma das tres variaveis acima esta ausente no backend ou na Vercel.

4. TypeDB: conectar a escrita do sistema

TypeDB e o banco de dados de textos e relacionamentos. Ele e necessario para login, posts e metadados usados para abrir a pagina publica do portfolio.

Nao crie um banco TypeDB vazio agora. O repositorio atual nao contem o schema TypeQL completo da rede social. Para funcionar, conecte o backend ao banco TypeDB existente, que ja possui o schema.
1 Receber dados do cluster existente

Solicite ao administrador os quatro valores:

TYPEDB_ADDRESS=https://endereco-do-cluster.typedb.com:80
TYPEDB_DATABASE=unigran_db
TYPEDB_USERNAME=admin
TYPEDB_PASSWORD=senha_do_cluster
2 O portfolio precisa encontrar

Ao publicar portfolio, o backend anota o post no TypeDB com estes metadados:

MetadadoPara que serve
portfolio-idIdentifica o case na URL publica.
portfolio-title / portfolio-summaryExibem titulo e descricao organizada.
portfolio-share-urlAbre /portfolio/usuario/case.
portfolio-document-urlAponta para PDF/DOC guardado no Supabase.
portfolio-media-urlAponta para imagem/video guardado no Cloudinary.
3 Testar sem conhecer TypeQL
  1. Com o backend ligado, abra http://localhost:3001/api/health. Deve aparecer "ok": true.
  2. Abra o frontend, registre ou entre em uma conta existente.
  3. Publique um portfolio e clique em Ver case completo.
  4. Se o case abrir apos atualizar a pagina, a gravacao TypeDB esta funcionando.
Se registrar/login ou publicar falhar: nao tente corrigir criando tabelas Supabase. Verifique primeiro credenciais e schema do TypeDB.

5. Variaveis no backend

As variaveis dizem ao backend onde estao os tres servicos. Para teste local, crie o arquivo backend/.env copiando backend/.env.example.

# TypeDB: textos e metadados
TYPEDB_ADDRESS=https://SEU_CLUSTER.typedb.com:80
TYPEDB_DATABASE=unigran_db
TYPEDB_USERNAME=admin
TYPEDB_PASSWORD=SUA_SENHA

# Login da aplicacao
JWT_SECRET=crie_uma_frase_longa_e_secreta
JWT_EXPIRES_IN=7d
PORT=3001
NODE_ENV=development
CLIENT_URL=http://localhost:5173
PUBLIC_APP_URL=http://localhost:5173
PUBLIC_PORTFOLIO_BASE_URL=http://localhost:5173

# Midias
CLOUDINARY_CLOUD_NAME=SEU_CLOUD_NAME
CLOUDINARY_API_KEY=SUA_API_KEY
CLOUDINARY_API_SECRET=SEU_API_SECRET

# Documentos
SUPABASE_URL=https://SEU_PROJECT_REF.supabase.co
SUPABASE_SERVICE_ROLE_KEY=SUA_SERVICE_ROLE_KEY
SUPABASE_DOCUMENTS_BUCKET=ava-entregas

# Nao usar Supabase para escrita nesta arquitetura
AUDIT_DATABASE_URL=

Frontend local

No desenvolvimento local nao e obrigatorio criar frontend/.env: o Vite ja redireciona /api e /portfolio para http://localhost:3001. Use variaveis apenas se precisar apontar para backend publicado.

# frontend/.env - somente se quiser testar contra backend publicado
VITE_API_URL=https://unigran-backend.vercel.app/api
VITE_PUBLIC_PORTFOLIO_URL=https://unigran.vercel.app
Nunca coloque chaves TypeDB, Supabase service_role ou Cloudinary secret no frontend.

6. Configurar na Vercel

Para corrigir o link publico em producao, as alteracoes de codigo precisam ser publicadas e o projeto backend precisa ter variaveis corretas.

Backend Variaveis privadas
  1. Abra o projeto unigran-backend na Vercel.
  2. Entre em Settings > Environment Variables.
  3. Adicione as variaveis da secao 5 para Production e, se usado, Preview.
  4. Salve e faca novo deploy do backend.
Frontend URLs publicas
  1. Abra o projeto frontend na Vercel.
  2. Defina VITE_API_URL=https://unigran-backend.vercel.app/api.
  3. Defina VITE_PUBLIC_PORTFOLIO_URL=https://unigran.vercel.app.
  4. Faca novo deploy do frontend.

7. Rodar localmente no Windows

Abra dois terminais PowerShell na pasta principal do projeto. O primeiro fica com backend aberto; o segundo, com frontend.

Terminal 1 Backend
cd "C:\Users\USUARIO\Desktop\REPESC\Vini\Unigran\backend"
Copy-Item .env.example .env
# Abra backend\.env e preencha as chaves reais antes de continuar
npm install
npm run dev

Quando estiver correto, o terminal mostra o backend em http://localhost:3001.

Terminal 2 Frontend
cd "C:\Users\USUARIO\Desktop\REPESC\Vini\Unigran\frontend"
npm install
npm run dev

Abra no navegador: http://localhost:5173.

Teste rapido do backend

Invoke-RestMethod http://localhost:3001/api/health

Resultado esperado: propriedade ok igual a True.

Teste completo do portfolio

  1. Abra o portal local e crie/login em uma conta.
  2. Publique um projeto no portfolio usando titulo, negrito e lista.
  3. Anexe PDF: valida Supabase.
  4. Anexe imagem/video em um post: valida Cloudinary.
  5. Clique em Ver case completo e atualize a pagina: valida TypeDB e a rota publica.

8. Erros comuns e correcao

Mensagem/sintomaCausa provavelO que fazer
Portfolio nao encontradoDeploy antigo ou metadado nao gravado no TypeDB.Publicar backend/frontend novos; testar republicacao de item antigo.
Supabase Storage nao configuradoVariavel ausente no backend.Preencher URL, service role e bucket; reiniciar backend/redeploy.
TenantNotFoundSUPABASE_URL incorreta.Copiar novamente Project URL em Settings > API.
Cloudinary nao configuradoCredencial ausente.Preencher as tres variaveis Cloudinary no backend.
Login/registro falha com erro internoTypeDB inacessivel ou schema ausente.Conferir cluster, banco, usuario e senha; usar o banco existente.
Frontend abre, API falha localBackend nao esta rodando em porta 3001.Iniciar npm run dev no diretorio backend.

9. Pendencia tecnica que precisa ser resolvida

Schema TypeDB nao esta no repositorio. O sistema depende do schema ja existente no cluster. Para nao ficar refem de configuracao manual, a equipe deve exportar ou criar o schema TypeQL versionado antes de montar outro ambiente.

Checklist final

  • Supabase possui apenas bucket ava-entregas para este fluxo.
  • Backend possui variaveis TypeDB, Supabase e Cloudinary.
  • Frontend nao recebeu nenhuma chave secreta.
  • Health check backend respondeu.
  • PDF anexado abre.
  • Imagem/video aparece.
  • Case publico abre e permanece aberto apos atualizar pagina.
